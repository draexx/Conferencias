Fundamentos de Control de Versiones

1. ¿Qué es Git?

No es una nube de archivos, es un registro de cambios.

Adiós al Caos

Olvídate de:

  • proyecto_final.zip
  • proyecto_final_v2_este_si.zip
  • proyecto_editado_Pedro.zip

Git guarda "fotografías" de tu progreso.

El Flujo Básico

Modificado

Haces cambios

Preparado

Seleccionas cambios

Confirmado

Guardas foto (Commit)

2. GitHub Desktop

Git sin necesidad de comandos (por ahora).

¿Por qué empezar con GUI?

  • Visualización clara de las diferencias (Diffs).
  • Gestión de historial con un clic.
  • Reducción de errores de principiante.

Conceptos Clave

  • Repository: La carpeta del proyecto controlada por Git.
  • Commit: El mensaje descriptivo de lo que hiciste.
  • History: La lista de todos tus pasos hacia atrás.

3. Mi Primer Repositorio

El Mensaje de Commit

Un buen mensaje explica qué y por qué:

[BIEN]: "Agrega tabla de sensores en bitácora"
[MAL]: "cambios", "asdf", "terminado"

Taller Práctico

  1. Instalar GitHub Desktop.
  2. Crear un repositorio local llamado Taller-Herramientas.
  3. Mover tu archivo BITACORA.md a esa carpeta.
  4. Realizar tu primer Commit con un mensaje descriptivo.
  5. Hacer un cambio y ver cómo Git detecta la diferencia.
